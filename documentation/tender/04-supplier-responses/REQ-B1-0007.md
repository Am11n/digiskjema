# Supplier Response: REQ-B1-0007 - Universal Design Compliance

## Requirement
**REQ-B1-0007**: Universal design compliance - WCAG 2.1 AA and EN 301 549

## Our Solution Approach

### WCAG 2.1 AA Compliance Implementation
We implement comprehensive WCAG 2.1 AA compliance through multiple layers:

**Technical Implementation:**
- Built-in accessibility checker integrated into development workflow
- Automated testing pipeline running axe-core accessibility scanner
- Manual accessibility audits conducted quarterly by certified auditors
- Continuous integration checks that block merges with accessibility violations

**Component Design:**
- All UI components built with accessibility-first principles
- Proper semantic HTML structure throughout the application
- Comprehensive keyboard navigation support
- Screen reader compatibility testing with NVDA, JAWS, and VoiceOver
- Color contrast ratios maintained at 4.5:1 for normal text (7:1 for large text)

**Features Implemented:**
- Skip links for main content navigation
- Proper heading hierarchy (H1-H6 structure)
- ARIA labels and landmarks for complex components
- Focus indicators visible on all interactive elements
- Alternative text for all images and icons
- Form labels properly associated with inputs
- Error messages programmatically associated with form fields

### EN 301 549 Compliance
European standard for ICT accessibility requirements:

**Procurement Compliance:**
- All digital interfaces meet EN 301 549 requirements
- Documentation available demonstrating compliance
- Regular third-party verification of accessibility standards
- User testing with people with disabilities conducted biannually

**Technical Standards Met:**
- Keyboard accessibility requirements
- Screen reader compatibility
- Alternative input methods support
- Cognitive accessibility considerations
- Visual design accessibility requirements
